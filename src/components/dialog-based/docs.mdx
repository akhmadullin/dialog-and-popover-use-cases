# Компоненты на основе элемента dialog

## Dialog
 
Как ни странно, на основе элемента `dialog` можно сделать компонент `Dialog`.

Он может иметь как встроенные фичи веб платформы, так и накрученный сверху кастом.

Но обо всем по порядку!

### Закрытие диалога на Escape

По дефолту браузеры дают возможность закрывать по нажатию на Escape только модальные диалоги.

Если эта фича необходима и для немодальных диалогов, то ее можно реализовать самим.

<Canvas of={CloseOnEscapeNonModalStories.CloseOnEscapeNonModal} />

### Закрытие диалога по клику на область за его пределами

По дефолту браузеры не дают такую возможность. Но такое поведение часто хотят видеть дизайнеры и бизнес. Его можно добавить самостоятельно.

<Canvas of={CloseOnOutsideClickStories.CloseOnOutsideClick} />

### Блокировка скролла страницы

Иногда, бывает полезно блокировать скролл остальной страницы, пока открыт диалог.

Элемент `dialog` не делает этого по дефолту. Но это можно исправить.

<Canvas of={PageScrollLockStories.PageScrollLock} />

## Confirm

На основе диалога можно делать вспомогательные компоненты - например, кастомизированный аналог метода `confirm()`.

Подобный компонент можно использовать там, где нужен свой кастомный дизайн и нет нужды блокировать исполнение основного потока во время показа `confirm`.
